You want to know when a competitor changes their pricing. So you build a monitor: fetch their pricing page every hour, send it to GPT-4o, ask "did anything change?" Simple.

Here's the problem: most of those fetches come back unchanged. A pricing page that updates once a week still generates 168 LLM calls a week. 167 of them say "nothing changed." You paid for all 167.

The right architecture: hash first, LLM second

The correct pattern separates fetching from reasoning:

  1. Fetch the page every run — you always need fresh data
  2. Hash the content and compare to the last stored hash — instant, free
  3. If unchanged → return cached insight, skip the LLM entirely
  4. If changed → extract the diff, send only the changed portion to the LLM

This gives you 100% recall (no changes missed) at a fraction of the cost.

Building it with Indra

Indra implements this pattern out of the box — including Bright Data integration to handle bot detection on competitor sites.

import indra
from anthropic import Anthropic

agent = indra.init(brightdata_api_key="your-bd-key")
client = Anthropic()

def analyze_change(goal, inputs, *_):
    msg = client.messages.create(
        model="claude-haiku-4-5",
        max_tokens=512,
        messages=[{"role": "user", "content": f"{goal}\n\nDiff:\n{inputs.get('diff', inputs.get('content', ''))}"}]
    )
    return msg.content[0].text

# Monitor a competitor's pricing page
result = agent.watch(
    url="https://competitor.com/pricing",
    question="Did any prices, tiers, or features change?",
    generation_fn=analyze_change,
)

print(result.changed)        # True / False
print(result.insight)        # LLM analysis if changed, cached insight if not
print(result.tokens_saved)   # tokens skipped this run

Monitoring multiple competitors

competitors = [

Running on a schedule

# cron: 0 * * * *  (every hour)
import indra
from anthropic import Anthropic

agent = indra.init(brightdata_api_key="your-bd-key")
client = Anthropic()

def analyze(goal, inputs, *_):
    return client.messages.create(
        model="claude-haiku-4-5", max_tokens=256,
        messages=[{"role": "user", "content": f"{goal}\n\n{inputs.get('diff', 'No change')}"}]
    ).content[0].text

result = agent.watch(
    url="https://competitor.com/pricing",
    question="Summarize any pricing changes",
    generation_fn=analyze,
)

# Runs hourly — LLM only fires when prices actually change

What Bright Data adds

Competitor sites actively block scrapers. Without a proper fetching layer:

  • Your IP gets banned after a few requests
  • JavaScript-rendered pricing tables return empty HTML
  • Geo-restricted pages show different prices in different regions

Indra uses Bright Data's Web Unlocker to handle all of this transparently — the same fetch call works on bot-protected, JS-heavy, and geo-restricted pages.
