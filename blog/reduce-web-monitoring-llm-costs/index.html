Web monitoring is uniquely wasteful in how it uses LLMs. Unlike a chatbot (one query per user interaction) or a batch job (one LLM call per item), a monitoring agent runs on a schedule — and most of its scheduled runs find nothing new.

If you're checking 20 pages every 2 hours, that's 240 fetches per day. If pages change on average once per day, 220 of those fetches confirm "nothing changed." At $0.05 per analysis, you're paying $11/day for nothing.

Two sources of waste

Waste 1: LLM called on unchanged pages. The biggest cost by far. Every fetch triggers an LLM call regardless of whether content changed. 80–95% of calls produce the same "no change detected" conclusion.

Waste 2: Full-page context on changed pages. When a page does change, most agents send the entire current page to the LLM. A 2,000-token pricing page might have changed by 3 numbers. The LLM reads 1,997 irrelevant tokens to find 3 relevant ones.

Eliminating Waste 1: hash-based change detection

import indra
from anthropic import Anthropic

agent = indra.init(brightdata_api_key="your-bd-key")
client = Anthropic()

def analyze(goal, inputs, *_):
    # inputs["diff"] is populated only when content changed
    content = inputs.get("diff") or inputs.get("content", "")
    return client.messages.create(
        model="claude-haiku-4-5", max_tokens=256,
        messages=[{"role": "user", "content": f"{goal}\n\n{content}"}]
    ).content[0].text

result = agent.watch(
    url="https://competitor.com/pricing",
    question="What pricing changed?",
    generation_fn=analyze,
)

# LLM is never called if the page hash matches the stored snapshot
print(f"Changed: {result.changed}")
print(f"Tokens saved: {result.tokens_saved}")

Eliminating Waste 2: diff-only context

When Indra detects a change, it passes inputs["diff"] to your function — the unified diff of what changed, not the full page. Typical diffs are 50–300 tokens vs 1,000–3,000 for the full page.
