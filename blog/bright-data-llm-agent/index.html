Building an LLM Agent with Bright Data — Web Intelligence That Scales

By Mahika Jadhav · May 31, 2026 · 6 min read

Building a web intelligence agent has two hard problems: getting the data and interpreting it efficiently. Most tutorials focus on the LLM side and wave away the fetching problem with requests.get(). In production, that breaks immediately — competitor sites block scrapers, JavaScript renders content dynamically, and geo-restrictions show different data to different IPs.

Here's how to solve both problems together: Bright Data for reliable data access, Indra for intelligent change-driven LLM routing.

What Bright Data solves

  • Bot detection: Rotating residential proxies make your fetches look like real users
  • JavaScript rendering: Web Unlocker executes JS before returning the page
  • CAPTCHA solving: Handled automatically
  • Geo-restrictions: Fetch from specific countries to see region-specific content

Without a proper fetching layer, your agent works on easy public pages and breaks on everything important.

What Indra adds on top

Bright Data solves fetching. But fetching reliably every hour still costs money if you're invoking the LLM on every fetch — even unchanged pages. Indra adds change detection on top of Bright Data: hash the response, compare to the last snapshot, and only route to the LLM when something actually changed.

Complete setup

pip install indra-ai
import indra
from anthropic import Anthropic

# Initialize with Bright Data credentials
agent = indra.init(
    brightdata_api_key="your-brightdata-key",
    brightdata_zone="web_unlocker1",  # your zone name
)

client = Anthropic()

def analyze(goal, inputs, *_):
    content = inputs.get("diff") or inputs.get("content", "")
    return client.messages.create(
        model="claude-haiku-4-5",
        max_tokens=1024,
        messages=[{"role": "user", "content": f"{goal}\n\n{content}"}]
    ).content[0].text

# Monitor a JS-heavy competitor site
result = agent.watch(
    url="https://competitor.com/pricing",
    question="Did pricing, features, or availability change? Summarize the impact.",
    generation_fn=analyze,
)

print(f"Changed: {result.changed}")
print(f"Insight: {result.insight}")
print(f"Tokens saved: {result.tokens_saved}")

SERP monitoring (search intelligence)

# Monitor search results for a query
result = agent.search_watch(
    query="your brand name review",
    question="Are there new negative reviews or competitor ads appearing?",
    generation_fn=analyze,
)

if result.changed:
    print(f"SERP changed: {result.insight}")

Monitoring with geo-targeting

# Check competitor pricing in different regions
regions = ["us", "uk", "de", "jp"]

for region in regions:
    result = agent.watch(
        url="https://competitor.com/pricing",
        question="What pricing is shown? Any regional differences from last check?",
        generation_fn=analyze,
        country=region,  # Bright Data fetches from this country
    )
    if result.changed:
        print(f"[{region.upper()}] Pricing change: {result.insight}")

Getting Bright Data credentials

  1. Sign up at brightdata.com
  2. Create a Web Unlocker zone (or use an existing proxy zone)
  3. Copy your API key and zone name
  4. Set BRIGHTDATA_API_KEY and BRIGHTDATA_UNLOCKER_ZONE env vars
import os
agent = indra.init(
    brightdata_api_key=os.environ["BRIGHTDATA_API_KEY"],
    brightdata_zone=os.environ.get("BRIGHTDATA_UNLOCKER_ZONE", "web_unlocker1"),
)
Bright Data offers a free trial with enough credits to test your monitoring setup. Indra's change detection means you'll use fewer Bright Data credits too — each fetch only triggers an LLM call if the content actually changed.
