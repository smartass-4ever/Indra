A production AI web monitoring agent needs to do four things well: fetch reliably (bypassing bot detection), detect changes efficiently (without calling the LLM on every fetch), analyze meaningfully (sending only diffs, not full pages), and alert on signal (not noise). Here's the complete build.

Architecture overview

Scheduler → Fetch (Bright Data) → Hash → Compare to snapshot
                                              ↓ changed?
                                        No → return cached insight
                                        Yes → extract diff → LLM → store insight → alert

Quick start with Indra

pip install indra-ai
import indra
from groq import Groq

agent = indra.init(brightdata_api_key="your-bd-key")
groq_client = Groq()

def analyze(goal, inputs, *_):
    content = inputs.get("diff") or inputs.get("content", "")
    return groq_client.chat.completions.create(
        model="llama-3.3-70b-versatile",
        messages=[{"role": "user", "content": f"{goal}\n\n{content}"}]
    ).choices[0].message.content

result = agent.watch(
    url="https://example.com/page",
    question="What changed? Is it significant?",
    generation_fn=analyze,
)

print(result.changed)
print(result.insight)
print(result.tokens_saved)

Multi-URL monitoring with different questions

WATCH_LIST = [
    {"url": "https://competitor.com/pricing",  "question": "Price or tier changes?"},
    {"url": "https://supplier.com/stock",       "question": "Stock or lead time changes?"},
    {"url": "https://regulator.gov/notices",    "question": "New enforcement actions or guidance?"},
    {"url": "https://news-site.com/tech",       "question": "New stories about AI or our market?"},
]

for item in WATCH_LIST:
    result = agent.watch(url=item["url"], question=item["question"], generation_fn=analyze)
    if result.changed:
        dispatch_alert(item["url"], result.insight)

SERP monitoring

queries = ["your brand name", "competitor name review", "best tool for X"]

for query in queries:
    result = agent.search_watch(query=query, question="New content or ranking changes?", generation_fn=analyze)
    if result.changed:
        print(f"SERP changed for '{query}': {result.insight}")

Running on a schedule (GitHub Actions)

# .github/workflows/monitor.yml
name: Web Monitor
on:
  schedule:
    - cron: '0 */4 * * *'  # every 4 hours
jobs:
  monitor:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - run: pip install indra-ai groq
      - run: python monitor.py
        env:
          BRIGHTDATA_API_KEY: ${{ secrets.BRIGHTDATA_API_KEY }}
          GROQ_API_KEY: ${{ secrets.GROQ_API_KEY }}
          SLACK_WEBHOOK: ${{ secrets.SLACK_WEBHOOK }}

Session analytics

stats = agent.get_stats()
print(f"Total fetches:      {stats['total_fetches']}")
print(f"Changes detected:   {stats['changes_detected']}")
print(f"LLM calls fired:    {stats['llm_calls_fired']}")
print(f"Tokens saved:       {stats['tokens_saved']:,}")
print(f"Cost saved:         ${stats['cost_saved_usd']:.4f}")
efficiency = 100 * (1 - stats['llm_calls_fired'] / max(stats['total_fetches'], 1))
print(f"Efficiency:         {efficiency:.0f}% LLM calls eliminated")

Production checklist

  • Set BRIGHTDATA_API_KEY for bot-protected sites
  • Choose LLM by cost: Groq (fastest/cheapest), Haiku (good quality/cheap), Sonnet (best quality)
  • Store state between runs — Indra uses SQLite by default, no config needed
  • Set up alerting: Slack webhook, email, PagerDuty, etc.
  • Schedule appropriately: competitive pages (2–4h), news (1h), regulatory (daily)
  • Monitor your own agent stats to track efficiency over time
