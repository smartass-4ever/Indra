How to Build a News Monitoring Agent in Python

By Mahika Jadhav · May 31, 2026 · 6 min read

News monitoring agents have a specific problem: news pages update frequently, but most fetches return the same articles you already saw. A tech news page might add one new story an hour — but checking every 15 minutes means 3 out of 4 fetches are redundant.

A naive LLM-powered news monitor pays for all 4 fetches. A smart one pays for 1.

The build

import indra
from groq import Groq

agent = indra.init(brightdata_api_key="your-bd-key")
groq_client = Groq()

def analyze_news(goal, inputs, *_):
    content = inputs.get("diff") or inputs.get("content", "")
    return groq_client.chat.completions.create(
        model="llama-3.3-70b-versatile",
        messages=[{"role": "user", "content": f"{goal}\n\n{content}"}]
    ).choices[0].message.content

# Monitor TechCrunch for AI news
sources = [
    {
        "url": "https://techcrunch.com/category/artificial-intelligence/",
        "question": "What new AI stories appeared? Summarize headlines and key points.",
    },
    {
        "url": "https://news.ycombinator.com",
        "question": "Any new top stories relevant to AI, LLMs, or developer tools?",
    },
    {
        "url": "https://openai.com/news",
        "question": "Did OpenAI announce anything new? Summarize.",
    },
]

for source in sources:
    result = agent.watch(
        url=source["url"],
        question=source["question"],
        generation_fn=analyze_news,
    )

    if result.changed:
        print(f"\n📰 NEW CONTENT: {source['url']}")
        print(result.insight)
        # send_to_slack(source["url"], result.insight)
    else:
        print(f"✓ No change: {source['url']} ({result.tokens_saved} tokens saved)")

Monitoring SERP news for brand mentions

# Watch for news about your brand or competitors
result = agent.search_watch(
    query="your-company-name announcement OR funding OR launch",
    question="Is there a new significant story about this company? What's the news?",
    generation_fn=analyze_news,
)

if result.changed:
    print(f"Brand mention: {result.insight}")

regulatory_pages = [

Building a digest

import datetime

daily_changes = []

for source in all_sources:
    result = agent.watch(url=source["url"], question=source["question"], generation_fn=analyze_news)
    if result.changed:
        daily_changes.append({"source": source["url"], "insight": result.insight})

if daily_changes:
    digest = "\n\n".join([f"**{c['source']}**\n{c['insight']}" for c in daily_changes])
    send_daily_digest(digest)

# Print stats
stats = agent.get_stats()
print(f"Sources checked: {stats['total_fetches']}")
print(f"New content found: {stats['changes_detected']}")
print(f"LLM calls: {stats['llm_calls_fired']} (vs {stats['total_fetches']} naive)")
print(f"Efficiency: {100*(1-stats['llm_calls_fired']/max(stats['total_fetches'],1)):.0f}% calls saved")
News pages often have dynamic elements (timestamps, view counts, ad banners) that change even when articles don't. Indra's hashing normalizes common noise — you get alerts on genuine content changes, not CSS updates.
