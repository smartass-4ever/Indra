You've built a web scraping agent powered by an LLM. It monitors pages, extracts insights, and sends alerts. It's working — but the cost is unsustainable. You're paying $0.03–0.15 per page analysis, running dozens of pages every hour, and 80% of those analyses conclude with "nothing changed."

The root cause is architectural: you're calling the LLM unconditionally on every fetch. The LLM doesn't know what the page looked like before. It can't distinguish "nothing changed" from "I haven't seen this page before." So it re-reads the whole thing every time.

The fix: change detection before LLM invocation

The correct architecture never asks the LLM "did this change?" — it answers that question with a hash comparison (free, instant) and only invokes the LLM to interpret changes that actually happened.

import indra
from groq import Groq

agent = indra.init(brightdata_api_key="your-bd-key")
groq_client = Groq()

def analyze_diff(goal, inputs, *_):
    # inputs["diff"] contains ONLY what changed — not the full page
    diff = inputs.get("diff", "")
    response = groq_client.chat.completions.create(
        model="llama-3.3-70b-versatile",
        messages=[{"role": "user", "content": f"{goal}\n\nWhat changed:\n{diff}"}]
    )
    return response.choices[0].message.content

# Monitor 50 pages — LLM only fires on the ones that changed
urls = load_urls_from_config()

for url in urls:
    result = agent.watch(
        url=url,
        question="Is this change significant? What action should we take?",
        generation_fn=analyze_diff,
    )

    if result.changed:
        alert_team(url, result.insight)

# Session summary
stats = agent.get_stats()
print(f"Fetches: {stats['total_fetches']}, LLM calls: {stats['llm_calls_fired']}")
print(f"Efficiency: {100 * (1 - stats['llm_calls_fired'] / stats['total_fetches']):.0f}% LLM calls eliminated")

The diff advantage

Even when a page changes, you don't need to send the whole page. A pricing page is 2,000 tokens of HTML — but the change might be 3 numbers. The diff is 40 tokens.

Indra extracts the unified diff automatically and passes it to your LLM function. Your LLM call goes from 2,000 tokens to 40–200 tokens on a typical change — an additional 95% reduction on top of the change detection savings.
